# CLAUDE.md — Webex AI Agent Live Interactive Demo

## Project Overview

This is a **pure static HTML/CSS/JavaScript** sales demo tool for Cisco Webex Contact Center AI capabilities. There is no build system, no package manager, no server-side code, and no external dependencies that need to be installed. Everything runs directly in the browser.

The demo is used by Cisco sales teams to showcase live, functional AI agent interactions across three industry verticals.

**Maintainers:** Jerad Cockroft (`jcockrof@cisco.com`) and Patrick Cornish (`pacornis@cisco.com`)

**Hosted on:** GitHub Pages at `https://jeradc.github.io/Webex-AI-Agent-Live-Interactive-Demo/`

## Architecture

### Two-Layer File Pattern

Each demo scenario follows a strict two-file pattern:

| Guide File    | App File    | Scenario              |
|---------------|-------------|-----------------------|
| `index.html`  | `app.html`  | Restaurante Superba   |
| `index2.html` | `app2.html` | Webex Bank            |
| `index3.html` | `app3.html` | Webex Healthcare      |

**Guide files (`index*.html`)** — The presenter-facing layer:
- Left sidebar with step-by-step navigation buttons (data-target attributes)
- Top bar with environment switcher buttons (data-env attributes) to jump between scenarios
- Main content area contains an `<iframe>` embedding the corresponding `app*.html`
- Multiple modal overlays triggered by sidebar navigation clicks

**App files (`app*.html`)** — The customer-facing simulated websites:
- Fully styled fictional websites (restaurant, bank, healthcare portal)
- Embed a live Webex Contact Center chat widget via `media.imi.chat/widget/js/imichatinit.js`
- Each app has unique `data-bind` and `data-guid` IDs tied to real Webex CC configurations

### Overlay/Modal System

Each guide file contains these modal overlays (toggled via `classList.add/remove("open")`):

| Overlay ID           | Content                                          | Triggered by nav button        |
|----------------------|--------------------------------------------------|-------------------------------|
| `overview-overlay`   | Demo script overview (auto-opens on load)        | Auto                          |
| `interact-overlay`   | QR codes for Voice, SMS, Webchat channels        | "Ways to Interact with AI Agent" |
| `ai-overlay`         | AI agent config screenshot carousel (4 slides)   | "AI Agent Configuration"      |
| `knowledge-overlay`  | Knowledge base data grid (hours, menus, policies)| "Knowledge Base Configuration"|
| `assist-overlay`     | Agent Assist screenshot                          | "Agent Assist"                |
| `help-overlay`       | Contact info for Jerad and Patrick               | "?" button                    |
| `image-zoom-overlay` | Full-screen zoom for `.zoomable-image` clicks    | Clicking any carousel image   |

### JavaScript Patterns

All JS is inline at the bottom of each HTML file — no external scripts, no modules. Key patterns:

- **Nav switching:** `data-target` attributes on sidebar buttons map to `id`s on `.instruction-panel` divs
- **Env switching:** `data-env` attributes on top buttons do `window.location.href = button.dataset.env`
- **Overlay toggling:** `.classList.add("open")` / `.classList.remove("open")` + `aria-hidden` attribute
- **Carousel:** Index-based with `aiIndex` variable, prev/next buttons, dot indicators
- **Image zoom:** Click any `.zoomable-image` → copies `src` to `#image-zoom-target` and opens overlay

## Development Workflow

### No Build Step Required

Open any file directly in a browser or serve with a simple HTTP server:

```bash
# Python
python3 -m http.server 8080

# Node (if available)
npx serve .
```

Then visit `http://localhost:8080/index.html`.

### How to Add or Modify a Demo Scenario

1. **To add a new scenario**, duplicate the `index.html` + `app.html` pair, rename them (e.g., `index4.html` + `app4.html`), then:
   - Update the `<title>` tag
   - Update the `data-env` buttons in all three (or more) index files to include the new entry
   - Update the `<iframe src="...">` in the new guide file
   - Replace the `data-bind` and `data-guid` in the new app file with valid Webex CC widget IDs
   - Update the simulated website content in the new app file

2. **To update demo content** (knowledge base, AI config slides, overlay text), edit the relevant overlay `<div>` sections in the appropriate `index*.html` file.

3. **To update QR codes**, change the `data=` query parameter in the `api.qrserver.com` URLs to actual phone numbers or SMS short codes.

4. **To add or replace images**, update the `img/` directory and update the `src` attributes in the `index*.html` files. Images are also served from GitHub Pages at `https://jeradc.github.io/Webex-AI-Agent-Live-Interactive-Demo/img/`.

### Keeping All Three Guide Files in Sync

The three `index*.html` files share the same structure, CSS, and JavaScript. They differ only in:
- The scenario name/branding in headings and text
- Which `data-env` button has `class="active"`
- The `<iframe src="...">` pointing to the correct `app*.html`
- The content in overlay cards (knowledge base data, QR codes, etc.)

When making structural changes (new overlays, new JS behavior, CSS changes), apply them to **all three** index files to keep them consistent.

## CSS Conventions

- CSS custom properties (variables) defined in `:root` at the top of each file's `<style>` block
- Color palette uses semantic names: `--bg`, `--panel`, `--text`, `--muted`, `--accent`
- Overlay visibility: controlled exclusively via `.open` class — never inline `display` styles
- Responsive breakpoints handled with a single `@media (max-width: 700px)` block at the end
- No CSS preprocessors — vanilla CSS only
- Transitions use `0.2s ease` consistently for overlay fades and hover effects

---

## Key Design Decisions

- **Self-contained HTML files:** No build tooling keeps deployment trivial (GitHub Pages, no CI/CD needed)
- **Iframe isolation:** The demo guide and the simulated app are separated so the chat widget in `app*.html` doesn't interfere with the guide UI
- **Auto-open overview:** `overviewOverlay.classList.add("open")` fires on script load, ensuring presenters see the script immediately
- **Aria attributes:** All overlays use `aria-hidden` toggled alongside the CSS class for accessibility
- **Image zoom:** Any image with `.zoomable-image` class gets a click-to-fullscreen behavior automatically

## What NOT to Do

- Do not add a build system, bundler, or package manager unless the complexity of the project genuinely requires it
- Do not extract inline CSS/JS to separate files without a compelling reason — the self-contained single-file approach is intentional for portability
- Do not modify `data-bind` or `data-guid` widget IDs without coordinating with Webex CC admins
- Do not commit `.DS_Store` or other OS artifacts
- Do not push to branches other than the designated feature branch without approval
